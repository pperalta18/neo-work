# Wall Graphics Production (AiKit Live) — Implementation Plan

## Summary

Produce **all 20 wall graphics** (vinyls / light-boxes) for the AiKit Live expo
3D maquette, and apply each to its wall at true scale, then export press-ready
CMYK PDF/X. This is **both system extensions and content production**, phased so
the tooling exists before the content scales, with the **hero piece as an early
vertical slice** that proves the code-track end-to-end.

Key decisions (see [spec](../specs/wall-graphics.md)):
- **Message-first curation** — one message per piece; cut any stat that doesn't make it land.
- **Two tracks** — code-rendered honest data-viz from a researched, sourced data file; image-gen only where imagery genuinely earns it.
- **Single wall registry** on `event-layout.json` (`invId N ↔ wall-(N-1)`).
- **3D** gains placement persistence, per-wall height, light-box + double-sided.
- **Nothing ships with unverified numbers.**

**Execution note:** once approved, the content-heavy phases (3–5) are intended to
run as a **Workflow (ultracode)** — fan out research, image-gen, and the 20 pages.
**Phase 4 (hero slice) is the recommended first executable milestone.** Phases 0–2
are code/tooling work done inline.
